# 1. 校园网

校园局域网可以说是所有智能设备间通信的渠道。目前已探明的包括但不限于希沃、电子班牌、监控、办公设备，以及搭建校园网本身的设备，如网关、防火墙、上网认证系统等。

## 结构

不同组、类型的设备通过各自的网关联系上局域网中心的设备。若要访问互联网地址或进行HTTPS连接，则要通过机制不明的身份验证，未通过的连接会被中断或跳转至**上网认证系统**。

## 硬件

涉及校园网核心的设备应该大多布置于远志楼5、6F的信息中心。通慧楼1F信息中心办公室中可能也有部分核心设备。外围的校园网设备中，常见的如H3C网关设备，一般设立在办公室内或教室走廊上（一般闲置）。

## 特性与研究

### 不完全的AP隔离

我们知道不同组、种类的设备可能在访问特定地址时受到不同程度的限制。但让人不解的是有许多例外：
- 山上的电子班牌将IPv4配置中调为自动IP时可访问校园网内网站，保持原本手动IP地址时则不能。山下的电子班牌没有这种区别。
- 一部分希沃可打开MYZX测速网站，另一部分则不能。

### 地理空间跨度

举例：电子班牌系统中的设备分布范围广泛；从校门口到每个班再到寝空门口。然而，它们的默认网关均为10.37.120.1。

### 限速

不同接入方式下的设备速度有差别。如同为办公室电脑，部分在MYZX测速网站中可取得1000 Mbps左右成绩，而部分仅有100 Mbps。

校园网内设备通过上网认证后访问互联网，存在严苛的、不同程度的限速。由于上网认证在同一账号登陆人数多时会提示错误而不立即下线，此时QQ却可正常登录使用，此时下载文件可达30MB/s。而当时同环境下正常认证后QQ下载速度仅3MB/s，可见限速之狠。

同为希沃，不同时间、地点的设备在访问相同资源时的速度均从500KB/s到5MB/s不等。
